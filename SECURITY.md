# Security Policy

[English](#english) | [Русский](#русский)

## English

## Coordinated Disclosure

If you discover a vulnerability in Umbrella Protocol, report it confidentially
before publishing details. We acknowledge receipt, triage the risk, and
coordinate the disclosure date.

## Contact

Email: `security@umbrellax.io`

PGP key not available for this release; please submit reports via `security@umbrellax.io` and the team will reply with a key for follow-up encrypted exchange.

If PGP is unavailable, send a short message without sensitive details and we
will agree on a protected channel.

## What To Include

- affected component, for example `umbrella-mls`, `umbrella-kt`, or FFI;
- short vulnerability description;
- reproduction steps or proof of concept;
- git commit or release version;
- impact assessment;
- whether you want public credit.

## Our Commitments

- acknowledgement within 72 hours;
- initial triage within 7 days;
- status updates at least every 14 days while the issue is open;
- coordinated disclosure after a fix or mitigation is available;
- public credit when the reporter wants it.

## Safe Harbor

Good-faith security research that follows this policy is permitted by the
repository license. Local code analysis, builds, tests, fuzzing, static
analysis, formal verification, timing measurements, cryptographic testing, and
local non-distributed modifications for vulnerability reporting are allowed.

This permission does not grant commercial use, public fork distribution,
operation of a public service, or attacks against third-party or production
infrastructure.

## Out Of Scope

- social engineering of users or staff;
- physical access to a user device without a protocol attack;
- vulnerabilities in third-party dependencies that already have an upstream fix;
- trivial volumetric denial of service.

## Advisories

After coordinated disclosure, advisories are published through GitHub Security
Advisories and the UmbrellaX security page.

## Русский

## Координированное раскрытие

Если вы нашли уязвимость в Umbrella Protocol, сообщите нам конфиденциально до
публичной публикации деталей. Мы подтверждаем получение, разбираем риск и
согласуем дату раскрытия.

## Контакт

Email: `security@umbrellax.io`

PGP-ключ для этого релиза не опубликован; присылайте сообщения на `security@umbrellax.io`, команда ответит с ключом для последующего зашифрованного обмена.

Если PGP недоступен, отправьте короткое письмо без чувствительных деталей, и мы
согласуем защищённый канал.

## Что включить

- затронутый компонент, например `umbrella-mls`, `umbrella-kt` или FFI;
- короткое описание уязвимости;
- шаги воспроизведения или proof of concept;
- git commit или версию выпуска;
- оценку влияния;
- нужно ли публичное упоминание автора отчёта.

## Наши обязательства

- подтверждение получения в течение 72 часов;
- первичный разбор в течение 7 дней;
- обновления статуса минимум раз в 14 дней, пока вопрос открыт;
- согласованное раскрытие после исправления или mitigation;
- публичное упоминание, если автор отчёта этого хочет.

## Safe Harbor

Добросовестная проверка безопасности в рамках этой политики разрешена лицензией
репозитория. Разрешены локальный анализ кода, сборка, тесты, fuzzing,
статический анализ, формальная проверка, тайминговые измерения,
криптографическое тестирование и локальные нераспространяемые изменения для
подготовки отчёта об уязвимости.

Это разрешение не даёт права на commercial use, публичное распространение
форка, запуск публичного сервиса или атаки на стороннюю либо
production-инфраструктуру.

## Вне области

- социальная инженерия пользователей или сотрудников;
- физический доступ к устройству пользователя без атаки на протокол;
- уязвимости в сторонних зависимостях, где уже есть upstream-исправление;
- простая объёмная атака отказа в обслуживании.

## Advisory

После согласованного раскрытия advisory публикуются через GitHub Security
Advisories и страницу безопасности UmbrellaX.
